Protocol

How L402 works.

No accounts. No API keys. No subscriptions. One HTTP round-trip and a Lightning payment is all it takes.

Make the request
Call any endpoint like a standard GET request. No headers, no auth. The server responds with HTTP 402 and a BOLT11 Lightning invoice in the response body.
Pay the invoice
Pay the invoice with any Lightning wallet — Phoenix, Breez, Alby, lncli. Payment settles in milliseconds. You receive a payment hash as proof.
Receive the data
Resend the same request with ?payment_hash= appended. The server verifies on-chain and returns the full JSON response.
Agents pay autonomously
AI agents using lnget or WebLN handle the full 402 → pay → retry loop automatically. No human needed. True machine-to-machine payments.
