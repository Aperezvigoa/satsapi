SatsAPI Documentation

SatsAPI is a Bitcoin market intelligence API that uses the Lightning Network L402 protocol for pay-per-call pricing. No registration, no API keys, no subscriptions. You pay in satoshis, you get data.

The API provides 8 endpoints covering real-time price data, mempool analysis, on-chain metrics, derivatives, AI-generated signals and full market summaries. Every response is structured for both human developers and autonomous AI agents.

Current status
SatsAPI is live. L402 payment enforcement is active — all endpoints require a Lightning payment to access.

Base URL

endpoint
https://satsapi.dev

All endpoints are served over HTTPS. HTTP requests are redirected to HTTPS.

Error Codes

Rate Limits

Rate limits are enforced per IP address. Limits vary by endpoint complexity:

TierEndpointsLimit
Standard/v1/price · /v1/mempool · /v1/tx · /v1/onchain · /v1/derivatives60 req/min
AI/v1/news · /v1/signal10 req/min
Heavy/v1/summary5 req/min

When a rate limit is hit, the response includes X-RateLimit-Limit, X-RateLimit-Remaining and Retry-After headers.

Authentication L402

SatsAPI uses the L402 protocol (HTTP 402 Payment Required) for authentication. Instead of API keys, you pay a tiny Lightning Network invoice per request. The entire payment cycle completes in under one second.

L402 is an open standard originally proposed by Lightning Labs. It enables machine-to-machine micropayments — AI agents can pay for API calls autonomously without human intervention.

No account required
There is no registration, no email, no credit card, no API key. The Lightning invoice is the credential.

Payment Flow

01

Make the initial request

Call any endpoint as a standard GET request. No headers required.

02

Receive HTTP 402 + invoice

The server responds with status 402. The WWW-Authenticate header contains a LSAT token and a Lightning invoice (BOLT11 format).

03

Pay the Lightning invoice

Use any Lightning wallet or client library to pay the invoice. Payment settles in milliseconds. You receive a payment preimage.

04

Repeat request with token

Send the same request with the Authorization: LSAT <token>:<preimage> header. The server verifies and returns the data.

Full L402 payment flow

javascript
async function callSatsAPI(path) {
  // 1. Initial request
  let res = await fetch(`https://satsapi.dev${path}`);

  // 2. Handle 402 — pay Lightning invoice
  if (res.status === 402) {
    const authHeader = res.headers.get('WWW-Authenticate');
    // authHeader format: "LSAT token=<token>, invoice=<bolt11>"
    const invoice  = authHeader.match(/invoice="([^"]+)"/)[1];
    const token    = authHeader.match(/token="([^"]+)"/)[1];

    // Pay with your Lightning wallet
    const preimage = await wallet.pay(invoice);

    // 3. Repeat with LSAT token
    res = await fetch(`https://satsapi.dev${path}`, {
      headers: { 'Authorization': `LSAT ${token}:${preimage}` }
    });
  }

  return res.json();
}

// Usage
const summary = await callSatsAPI('/v1/summary');
console.log(summary.data.bot_ready.signal); // "HOLD"

Python

python
import requests

def call_sats_api(path: str, wallet=None) -> dict:
    """Call SatsAPI with optional L402 payment handling."""
    url = f"https://satsapi.dev{path}"

    # 1. Initial request
    r = requests.get(url)

    # 2. Handle 402 payment
    if r.status_code == 402 and wallet:
        auth = r.headers.get("WWW-Authenticate", "")
        invoice = re.search(r'invoice="([^"]+)"', auth).group(1)
        token   = re.search(r'token="([^"]+)"', auth).group(1)

        # Pay invoice
        preimage = wallet.pay(invoice)

        # 3. Retry with token
        r = requests.get(url, headers={
            "Authorization": f"LSAT {token}:{preimage}"
        })

    r.raise_for_status()
    return r.json()

# Usage
data = call_sats_api("/v1/signal")
print(data["data"]["signal"])      # "HOLD"
print(data["data"]["confluence"])  # 52

AI Agent Integration

SatsAPI is designed from the ground up for autonomous AI agents. The L402 protocol lets agents pay for data without human intervention, and the bot_ready object eliminates all parsing complexity.

The bot_ready Object

Every call to /v1/summary (and /v1/signal) returns a flat bot_ready object at the top level of the response. All fields are primitives — no nested objects, no arrays, no post-processing.

Zero parsing required
if (data.signal === 'BUY' && data.confluence > 65 && data.leverage_risk !== 'EXTREME') — that's the entire decision logic.

Autonomous Payment Flow

Using lnget, the payment is handled transparently:

bash
# lnget handles 402 → pay → retry automatically
lnget https://satsapi.dev/v1/summary | jq .data.bot_ready

async function tradingCycle() {
  const { data } = await (await fetch("https://satsapi.dev/v1/summary")).json();
  const b = data.bot_ready;

  // Only act on high-confluence signals
  if (b.signal === "BUY" && b.confluence >= 65 && b.leverage_risk !== "EXTREME") {
    await exchange.createOrder("BTC/USDT", "buy", b.price_usd * 0.995);

  }

    await exchange.closePositions();

}

cron.schedule("0 */4 * * *", tradingCycle);
